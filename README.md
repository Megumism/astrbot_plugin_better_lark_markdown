# astrbot-better-lark-markdown

这是一个面向飞书的增强版 Markdown 解析器插件，解决表格、图片显示等问题，目标是以最小的改动支持更多的 Markdown 语法和功能。

> 注意，本插件不作用于“流式输出”，因为在使用 CardKit 流式卡片时，大部分Markdown语法都能得到正确解析，如果流式输出没有问题，建议使用流式输出。（但实测如果模型输出有图片链接，很大概率导致流式更新文本失败，吞没后面所有消息。）
> 是否启用“分段发送”消息对插件部分设置也有影响。

## 1. 目前改进支持的 Markdown 语法

所有飞书原生支持的 Markdown 语法（详见下文飞书原生支持的 Markdown 语法列表）外，本插件增加了以下支持：

- 表格
    - 转换为飞书卡片发送，卡片原生支持表头对齐
- 下划线（`<u>文本</u>`：<u>文本</u>）
- 任务列表（）
    - `- [ ] 待办事项` 转换为 `- ⬜ 待办事项`
    - `- [x] 已完成事项` 转换为 `- ✅ 已完成事项`
- 水平分割线
    - 除`---`、`***`外，添加了对`- - -`和`* * *`的支持
- 图片
    - 支持图片链接直接显示为图片（飞书原生仅支持部分图片链接显示为图片）
    - 支持图片链接显示为图片的前提是链接必须以`http://`或`https://`开头，并且链接的末尾必须是图片格式的扩展名（如`.jpg`、`.png`、`.gif`等）。如果链接不符合这些条件，飞书将无法正确解析并显示图片。

## 2. 插件配置项

如果开启了原生的分段发送，再加上本插件的分块发送，消息会被打碎成很多很小的气泡发送，每个气泡开头如果都对用户原消息进行引用，则会显得冗余臃肿，因此增加了一个配置项来控制飞书消息的发送方式：

| 可选值 | 说明 |
|--------|------|
| `auto` | 仅当回复内容不同于上一次时采用回复形式发送 |
| `reply` | 所有消息回复发送（AstrBot 默认行为） |
| `direct` | 所有消息直接发送 |
